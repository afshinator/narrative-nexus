### V1b. Agent 1 clustering logic

**File:** `pipeline/agent1_intake.py:97-104`

```python
matrix = np.array(embeddings, dtype=np.float64)
matrix_norm = normalize(matrix)  # unit vectors for cosine distance

clustering = DBSCAN(
    eps=0.5,            # maximum cosine distance for neighbors
    min_samples=2,      # minimum articles to form a core cluster
    metric="cosine",
).fit(matrix_norm)
```

- **Similarity metric:** Cosine distance on normalized embedding vectors (384-dim from all-MiniLM-L6-v2).
- **Threshold:** eps=0.5 cosine distance.
- **New article joining:** DBSCAN assigns each article a label. Articles with label=-1 (noise) get their own singleton cluster (`agent1_intake.py:138-141`). Non-noise labels share a cluster. There is no incremental "add to existing cluster" path — all articles are re-clustered from scratch in each run.
- **Input filtering:** Only articles with `body_status='AVAILABLE'` AND no existing claims are clustered (`agent1_intake.py:65-75`).

## V3: R_VAL FORMULA

### V3a. R_val computation

**File:** `pipeline/snapshots.py:48-85`

```python
def compute_r_val_raw(conn, *, as_of=None) -> dict[int, float | None]:
    # Numerator: absorbed claims where source was first reporter
    absorbed = count of claims WHERE c.state = 'CONSENSUS_ABSORBED'
               AND cs.first_seen_at = MIN(first_seen_at) for that claim
               AND c.created_at <= as_of (if provided)
    
    # Denominator: originated claims (same as R_orig)
    originated = compute_r_orig_raw(conn, as_of=as_of)
    
    # Formula:
    for sid, orig in originated.items():
        abs_count = absorbed.get(sid, 0)
        result[sid] = abs_count / orig if orig > 0 else None
```

**Formula:** `R_val = absorbed_first_reporter_claims / total_first_reporter_claims`

- **Numerator:** Only CONSENSUS_ABSORBED claims where this source was the first reporter.
- **Denominator:** ALL claims where this source was the first reporter (regardless of state — PENDING, ABSORBED, or UNRESOLVED are all counted in `compute_r_orig_raw`).
- PENDING claims ARE in the denominator but NOT in the numerator.

### V3b. R_orig — "first reporter" definition

**File:** `pipeline/snapshots.py:15-45`

```sql
SELECT cs.source_id, COUNT(*) as cnt
FROM claim_sources cs
INNER JOIN (
    SELECT claim_id, MIN(first_seen_at) as first_seen
    FROM claim_sources
    GROUP BY claim_id
) firsts ON cs.claim_id = firsts.claim_id
    AND cs.first_seen_at = firsts.first_seen
GROUP BY cs.source_id
```

A source is the "first reporter" of a claim if its `first_seen_at` in `claim_sources` equals the MINIMUM `first_seen_at` across ALL sources for that claim. Since `claim_sources` has one row per (claim_id, source_id), this is: for each claim, which source reported it earliest? If multiple sources share the same minimum timestamp, they ALL count as first reporters (tie).
