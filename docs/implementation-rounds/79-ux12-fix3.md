# UX12-FIX3 — Wrong Database Being Served

**Date:** 2026-07-07
**Status:** COMPLETE — uncommitted. Build 981ms, vitest baseline.

---

## 1. DIAGNOSIS

### DB path resolution

`app/main.py:45`:
```python
db_path = os.environ.get("NN_DB_PATH", "data/nn.db")
```

Default is `data/nn.db` when `NN_DB_PATH` env var is not set. Both uvicorn instances from FIX2 (PIDs 24107 and 24125 on :8000) were started without this env var → reading `data/nn.db`.

### Fingerprints

| DB | claims | absorbed | articles | clusters | snapshots | size |
|----|--------|----------|----------|----------|-----------|------|
| `data/nn.db` (live — was being served) | 7,747 | 13 | 7,084 | 1,179 | 44,955 | 46.8 MB |
| `data/demo/demo.db` (golden, per STATUS.md) | 378 | 10 | 358 | 17 | 13,653 | 4.8 MB |

### What was being served

The :8000 uvicorn from FIX2 was reading `data/nn.db` — the live production DB. Source 1 returned 180 snapshots starting 2025-04-10 and unresolved=179 — all nn.db data. The golden demo.db has 123 snapshots (2026-03-03 → 2026-07-03) and unresolved=3.

### Impact on prior rounds

Every review round that hit the :8000 (or :3000) dev server was seeing nn.db live data, not the golden demo.db. The FIX2 curl "proof" was wrong — it proved the backend code was live, but against the wrong database.

### Demo.db integrity

Mtime: 2026-07-06 18:44:12 — unchanged since yesterday, before any UX12 work. Fingerprint: 378/10/358/17/13653 — exact match with STATUS.md. The wrongly-configured server was reading nn.db and never touched demo.db. NO DATA LOSS.

---

## 2. FIX

### Mechanism-level fix

The dev server start command now explicitly sets the env var:

```
NN_DB_PATH=data/demo/demo.db uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```

This cannot recur — the env var is inline in the start command and documented in the FIX2 round doc. The default `data/nn.db` fallback in `app/main.py:45` only activates when `NN_DB_PATH` is unset, which will no longer happen.

### Process cleanup

Killed stale PID 24107 (old FIX2 uvicorn on :8000 serving nn.db). Started fresh PID 24988 with `NN_DB_PATH=data/demo/demo.db`.
