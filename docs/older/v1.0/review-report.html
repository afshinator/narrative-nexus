// PRIORITY ISSUES
P1

Archetype colors collide with polarity colors

Early Breaker = green, Noise Generator = red, Selective but Accurate = blue, Consensus Follower = amber. These overlap directly with the green/amber/red polarity system. A green Early Breaker dot in the bottom-right quadrant reads as positive even when the source has low validation. An amber Consensus Follower dot in the top-right quadrant reads as a warning when it isn't one.

FIX: Use four hues with equal lightness that don't overlap with the polarity palette — a desaturated set (blue-cyan, purple, teal, rose) that only appears on badges and scatter plot dots, never on metric cells.
P1

Scatter plot renders above the table on the leaderboard

The leaderboard is the primary scanning surface. Users arrive to compare sources across 6 metrics. The scatter plot is supplementary — it visualizes the same data. Stacking it first means users scroll past a large chart every visit. The table — the actual working instrument — is below the fold.

FIX: Move the table above the scatter plot, or use a side-by-side layout on wider screens so both are visible simultaneously.
P2

Interactive elements lack states

Focus states are absent throughout — tabbing reveals no visible ring. Timeline dots have title attributes but no click behavior (the brief describes clicking to navigate). Chart.js tooltips show data but the scatter chart's onClick only works when hitting a data point exactly. No loading, empty, or error states exist for any chart container.

FIX: Add visible focus rings (2-3px, 3:1 contrast). Make timeline dots clickable to coverage analysis. Implement loading/empty/error states for each chart. Broad scatter chart click target area.
P2

Body type runs small without a consistent scale

Metric labels at 0.75rem, body at 0.85rem, table cells at 0.82rem, nav at 0.8rem. Sizes appear chosen independently rather than from a scale. The smallest sizes on dark backgrounds with mid-gray text may dip near WCAG AA contrast limits.

FIX: Define a 3-step type scale (e.g., 0.75/0.875/1rem for body, 1.25rem for headings). Increase body minimum to 0.875rem. Verify all text combos against WCAG AA 4.5:1.
P2

Stub pages communicate nothing useful

The 4 stub pages (Pipeline Flow, Ad-Hoc, Panel, Onboarding) show "Contents TBD" with a description but no structural skeleton. Landing on a blank centered box with text feels broken rather than "coming soon." The muted nav styling is correct, but the pages themselves need layout bones.

FIX: Give each stub a minimal page frame — show the eventual layout shape (left panel, content area, search bar skeleton, etc.) even if all interior content is placeholder.
